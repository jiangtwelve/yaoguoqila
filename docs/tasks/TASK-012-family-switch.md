# TASK-012: 家庭切换与家庭管理

## Objective
实现家庭功能统一入口，包括家庭切换和家庭管理（含成员角色权限），让多家庭场景形成可用闭环。

## Scope
- 首页家庭名称升级为可点击的家庭芯片，打开家庭中枢面板。
- 家庭中枢面板展示当前用户已加入的所有家庭，支持切换当前家庭。
- 家庭中枢面板提供创建新家庭入口（复用已有 GlassModal），创建成功后自动切换到新家庭并返回首页；取消创建时回到家庭中枢面板。
- 家庭管理视图（面板内切换）：
  - 重命名家庭（复用 GlassModal）。
  - 查看成员列表，展示角色标识（管理员 / 普通成员）。
  - 管理员可移除普通成员。
  - 管理员可解散家庭（红色确认弹窗）。
  - 普通成员可退出家庭（红色确认弹窗）。
- 家庭名称创建和重命名最多 12 个字。
- 成员角色规则：
  - 家庭创建者默认为管理员。
  - 管理员和普通成员对物品的操作权限一致。
  - 管理员退出家庭前需先转移管理员身份或解散家庭。
- 切换家庭后刷新首页物品、位置和统计。
- 保留无家庭状态下的创建入口。

## Non-Scope
- 不实现邀请码、加入家庭功能（归入 TASK-014 独立实现）。
- 不实现邀请码分享页面。
- 不实现复杂成员权限细分（除管理员/普通成员外）。
- 不实现管理员转移功能（v0.1 暂不支持）。

## Acceptance Criteria
- 用户可从首页家庭芯片打开中枢面板，看到所有已加入家庭。
- 当前家庭有明确选中状态，打开家庭中枢时列表自动滚动到当前家庭。
- 点击其他家庭可切换。
- 切换后首页名称、物品列表、位置和统计同步变化。
- 家庭管理视图展示成员列表和角色标识。
- 管理员可重命名家庭、移除成员、解散家庭。
- 普通成员可重命名家庭、退出家庭。
- 解散家庭后回到家庭中枢列表；若解散当前家庭则刷新首页数据，若无任何家庭则进入无家庭状态。
- 操作失败时有清晰错误提示。
- 家庭中枢内创建新家庭成功后自动切换到新家庭并返回首页；取消创建时回到家庭中枢。
- 无家庭状态下首页保持创建家庭引导。

## Cloud Function Actions
- `family.switchFamily` — 切换当前家庭，更新 `currentFamilyId`。
- `family.renameFamily` — 重命名家庭。
- `family.getMembers` — 获取成员列表（含角色）。
- `family.removeMember` — 管理员移除成员。
- `family.leaveFamily` — 普通成员退出家庭。
- `family.dissolveFamily` — 管理员解散家庭。
