# Roadmap

## 生命周期原则
- 项目不再只按“下一个任务”滚动推进，而是按版本目标推进。
- 每个任务必须归属到一个 release 或 phase。
- 当前 release 的退出标准满足时，agent 应主动提醒用户进入版本级验收，而不是继续无边界加功能。
- 新需求先判断归属：当前 release 必需、后续 release、或 Backlog。

## Release Overview

### v0.1 测试版闭环
- Status: `active`
- 目标：在微信小程序中形成可完整体验的家庭物品保质期管理闭环。
- 目标用户：项目发起人和少量内部测试用户。
- 关键完成线：真实云开发环境下，用户可以设置昵称、创建/加入家庭、切换家庭、管理物品、多图上传、查看详情、编辑、标记用完、删除，并完成基础手动验收。
- 详情文件：[docs/releases/v0.1.md](releases/v0.1.md)

### v0.2 提醒能力版
- Status: `planned`
- 目标：接入真实临期/过期提醒能力。
- 包含方向：微信订阅消息模板、定时扫描云函数、通知发送记录、防打扰策略、失败重试。
- 不阻塞 v0.1 测试版。

### v0.3 家庭协作增强版
- Status: `planned`
- 目标：完善多人家庭协作能力。
- 包含方向：成员角色、权限细分、邀请管理、成员移除、家庭退出/解散。
- v0.1 只实现最小加入家庭能力。

### v1.0 首个正式版
- Status: `planned`
- 目标：达到可对外发布的稳定版本。
- 包含方向：稳定性修复、基础运营配置、隐私与数据说明、发布准备、关键端到端回归。

## Backlog Parking Lot
- Android/iOS 真实推送。
- 条码识别。
- OCR 识别日期和保质期。
- AI 菜谱或消耗建议。
- 多语言。
- 大数据量列表优化会在 v0.1 后评估是否进入 v0.2/v0.3，当前不阻塞测试版闭环。
