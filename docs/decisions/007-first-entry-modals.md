# Decision 007: 首次进入昵称弹窗与无家庭空状态

## Context
当前小程序开发无法默认获取用户真实微信昵称，因此用户首次进入时需要手动设置昵称。同时，用户没有家庭时不一定要立即创建家庭，也可能接受他人的邀请加入家庭。

## Decision
MVP 首次进入流程：
- 未设置昵称时，在首页弹出设置昵称弹窗。
- 无家庭时，在首页展示空家庭状态，引导用户创建家庭或加入/接受邀请。
- 用户主动点击创建家庭后，打开创建家庭弹窗。
- 设置昵称弹窗和创建家庭弹窗都只包含一个输入框和确认操作。
- 不为设置昵称或创建家庭单独设置页面。

## Consequences
- 首页 Design Anchor 需要覆盖首次昵称弹窗、无家庭空状态、创建家庭弹窗和正常列表状态。
- 用户模型需要记录昵称是否已设置，例如 `hasSetDisplayName`。
- 家庭切换中的“创建新家庭”也应打开创建家庭弹窗，而不是进入独立页面。
