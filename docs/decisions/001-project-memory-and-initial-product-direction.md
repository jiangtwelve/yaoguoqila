# Decision 001: 项目记忆与初始产品方向

## Context
用户希望制作一个名为「要过期啦」的应用，支持微信小程序、Android、iOS 三端通用。核心功能是用户录入物品名称、保质期、过期时间、图片、位置、备注等信息，并在物品即将过期或已经过期时推送提醒。物品应按家庭单位分类。

## Decision
- 建立项目连续性文档作为长期协作的单一事实来源。
- 产品初始方向记录为「家庭单位下的物品保质期管理和临期/过期提醒」。
- MVP 先围绕家庭、物品、提醒、列表筛选和首页摘要展开。
- MVP 协作深度选择「个人家庭轻协作」：首版保留家庭作为组织单位，但不实现完整多人邀请、审批和权限管理。

## Consequences
- 后续 agent 可以通过 AGENTS.md、docs/handoff.md 和 docs/tasks.md 继续工作。
- 具体 MVP 协作深度、技术选型和提醒渠道仍需用户确认后才能进入实现阶段。

## Open Questions
- 是否确认跨端框架优先？
- 首版是否必须接入真实推送？
