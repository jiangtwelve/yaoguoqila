# Decision 004: MVP 单首页导航

## Context
原页面地图建议使用底部 Tab：首页、物品、我的。用户反馈首版没有必要做三个页面，希望一个首页展示所有物品，并提供查询、新增和家庭切换。

## Decision
MVP 使用单首页作为主界面，不使用底部 Tab。

首页需要包含：
- 当前家庭名称。
- 点击家庭名称切换家庭。
- 当前家庭下所有物品。
- 物品按剩余保质期排序。
- 查询功能。
- 新增按钮，用于录入新物品。
- 不做状态筛选，默认按剩余保质期升序排序。

## Consequences
- 家庭首页仍是 Design Anchor，但它同时承担物品列表能力。
- 独立「物品」Tab 和「我的」页不进入 MVP。
- 后续原型应优先打磨首页的信息密度、排序、搜索、新增入口和家庭切换体验。
