# UI

## 设计目标
- 产品应给人「清爽、可靠、家用、有行动优先级」的感觉。
- 信息密度要适合小屏：临期和过期事项优先可见，避免装饰压过任务。
- 面向家庭日常使用，文案简洁温和，不制造焦虑。

## 初步视觉规则
- 首页优先突出「需要处理的东西」，而不是营销式大标题。
- 临期、已过期、正常三类状态要有清晰视觉区分。
- 卡片圆角不超过 8px，除非最终设计系统确认不同规则。
- 按钮优先使用明确图标或图标 + 文字。
- 表单控件要大拇指友好，适配小程序和移动端触摸。
- 不使用单一色相铺满全局；状态色应服务信息识别。

## 初步交互规则
- 新增物品应从首页和物品列表都能快速进入。
- 物品状态筛选应一键可达，不藏在深层菜单。
- 删除物品需要二次确认。
- 同类 UI 或交互在 2 处及以上出现，并且需要保持同一视觉语言或后续会统一微调时，应主动抽取组件或共享样式；若暂时不抽取，需要在 dev-log/handoff 记录原因和后续触发条件。
- 设置昵称、创建家庭、新增风险确认等轻量弹窗统一使用 `GlassModal` 组件；后续类似弹窗默认沿用这一套遮罩、玻璃卡片、标题、正文和双按钮结构，除非有明确的新 UI 验收。
- 需要用户输入的 `GlassModal` 必须让操作目标一眼可见，但不能通过堆叠说明文字来实现；默认采用标题 + 显眼大输入槽 + 操作按钮的极简结构，输入槽需要有足够高度、亮度、边界和占位文本，让用户第一眼看到可输入区域。
- `GlassModal` 遮罩应保持轻薄，让用户能看清蒙版后的页面层级；当前遮罩以低透明冷暖光感层叠加 `rgba(16,20,24,0.12)` 和轻模糊表达，不使用厚重暗幕。
- 新增物品若保存后会立即成为临期、今天到期或已过期状态，需要在创建前二次确认，避免用户误录入错误日期；使用统一 `GlassModal`，标题显示「提醒」，提示文本居中，按钮为 `取消` / `确认`。临期提示缩短为「还有 x 天到期，确认添加吗？」，其中剩余天数使用临期橙胶囊高亮；今天到期提示为「今天到期，确认添加吗？」，已过期提示为「已过期，确认添加吗？」。
- 小程序原生确认弹窗按钮文案必须控制在 4 个字符以内，避免 `showModal` 参数校验失败；需要定制样式的确认弹窗优先使用项目内 `GlassModal`。
- 保存失败要保留用户已输入内容。
- 图片为可选项，不能阻塞保存。
- 页面级初始加载统一使用骨架屏，骨架形态应贴近当前页面真实结构；不要使用空白页中间单独 spinner 或 loading 文案。
- 短操作反馈可使用 toast、顶部轻浮层或按钮内状态，例如下拉刷新；新增/编辑物品保存属于阻断性操作，保存中需要使用全屏轻玻璃 loading 明确锁定页面，不只依赖按钮内状态。不要用短操作反馈替代页面级骨架屏。
- 后续任何样式微调若影响页面视觉、按钮状态、加载反馈、弹窗或表单层级，必须同步更新本文档对应规则或视觉方向说明。

## Accepted Baseline
- 当前 Design Anchor「流光毛玻璃」已由用户确认作为可继续开发的 UI 基线。
- 已验收范围：首页玻璃摘要层、玻璃搜索框、玻璃列表、状态标签、新增表单、无家庭空状态、昵称/创建家庭弹窗；新增风险提醒弹窗仍需用户按本次调整视觉验收。
- 后续仍允许进行样式微调；微调需要保持当前基线的清透、轻盈、可读和家用日常感，并在本文件同步记录。

## Visual Direction
- Tone: 流光毛玻璃、轻盈、未来感、清透但仍适合家庭日常。
- Palette: 冷暖混合浅色背景，石墨青/雾青/淡紫低透明度光感层，玻璃白 `rgba(255,255,255,0.28-0.74)`，主文字 `#101418`，强调石墨青 `#4f7f78`，过期红 `#e5483f`，临期橙 `#ff9500`。
- Typography: 使用系统字体栈，标题加重但不夸张；通过透明层级、模糊和空间感建立风格，而不是靠强字体。
- Layout: 首页使用浮动玻璃摘要层 + 玻璃搜索框 + 半透明玻璃列表；新增表单使用玻璃录入台，每个字段都是独立玻璃浮层。
- Controls: 新增入口为深色半透明悬浮按钮；搜索框、状态、历史位置、保存按钮都使用统一毛玻璃材质。新增/编辑页固定保存按钮在可用时使用深石墨玻璃，在禁用时使用去饱和灰玻璃、弱阴影和灰色文字，明确表达当前不可保存；未满足保存条件或表单信息未准备好时，点击保存按钮需要给出 toast/错误提示，不能静默无响应。表单加载中或保存中必须锁定图片、输入框、日期选择器、历史位置、备注等其他表单操作，不能允许用户继续改动待保存数据。小程序端固定保存按钮需要使用 `cover-view` 并在保存入口收起键盘；保存中使用全屏轻玻璃 `cover-view` loading，包含 spinner、保存标题和同步说明，同时覆盖原生 `textarea` 等组件与底部保存按钮，避免保存中继续触发页面操作。
- Loading: 首页、详情页、编辑页使用浅玻璃骨架屏语言，骨架块以低透明白、细边框、柔和 shimmer 表达加载中；首页骨架模拟 dashboard/search/list，详情页骨架模拟图片/日期/信息，编辑表单骨架模拟图片区、字段和日期模块。新增表单不显示骨架屏，应直接呈现空白录入表单，并在表单选项加载完成后补充历史位置。
- Reminder summary: 首页顶部 glass dashboard 展示需要留意总数，下方用两个玻璃指标块展示「已过期」和「临期」。
- Item rows: 首页物品保留单列列表，但容器和缩略块都使用毛玻璃层；左侧优先显示物品图片，没有图片时显示统一占位图标，中间为名称和「位置 · 到期日」，右侧为状态标签。若用户未设置位置，只显示到期日。
- Item form: 表单顺序为图片、名称、日期、位置、备注；名称和日期必填，必填星号使用红色。日期必须使用日期选择器，不允许自由输入。生产日期 + 保质期区域使用左右公式式玻璃卡片表达，去掉卡片内重复标签，输入内容垂直居中；保质期数值占位使用稳定文本层，避免原生输入框 placeholder 在首次布局时偏移。预计到期结果根据状态变色：正常用主色，临期/今天到期用橙色，已过期用红色。位置是自由输入框，历史位置仅作为快捷选择。分类暂不显示。提醒不在 MVP 表单中配置，使用系统默认提醒策略。
- WeChat safe area: 首页顶部必须适配状态栏、刘海屏和微信右上角胶囊操作栏。胶囊所在行的左侧空间可用于家庭名称/标题，右侧必须预留胶囊空间；不要为了避让胶囊把整行都空出来。
- Do not: 不使用强卡通、粗描边、多色便签、复杂插画、营销式 hero、过度拟物化厨房元素；毛玻璃应克制，避免变成紫蓝渐变模板。
