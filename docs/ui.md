# UI

## 设计目标
- 产品应给人「清爽、可靠、家用、有行动优先级」的感觉。
- 信息密度要适合小屏：临期和过期事项优先可见，避免装饰压过任务。
- 面向家庭日常使用，文案简洁温和，不制造焦虑。

## 初步视觉规则
- 首页优先突出「需要处理的东西」，而不是营销式大标题。
- 临期、已过期、正常三类状态要有清晰视觉区分。
- 卡片圆角不超过 8px，除非最终设计系统确认不同规则。
- 按钮优先使用明确图标或图标 + 文字。
- 表单控件要大拇指友好，适配小程序和移动端触摸。
- 不使用单一色相铺满全局；状态色应服务信息识别。

## 初步交互规则
- 新增物品应从首页和物品列表都能快速进入。
- 物品状态筛选应一键可达，不藏在深层菜单。
- 删除物品需要二次确认。
- 保存失败要保留用户已输入内容。
- 图片为可选项，不能阻塞保存。

## Pending Acceptance

- 待验收范围：顶部家庭名称、日期、加号按钮、临近提醒数字、查询框、物品列表、状态文字、无家庭空状态、昵称/创建家庭弹窗。

## Draft Visual Direction
- Tone: 极简、克制、精致。
- Palette: 近白绿色背景 `#f4f6f1`、墨色文字 `#151713`、灰绿色辅助字 `#596154`、陶土色提醒 `#8c5d4a`、深红过期 `#8d2c22`。
- Typography: 中文标题优先使用宋体气质字体栈 `Songti SC` / `STSong`，正文保持平台默认 sans 字体，形成克制对比。
- Layout: 首页使用大留白、细线分隔、无底部 Tab、无卡片堆叠；物品列表以横向行和细分隔线承载信息。
- Controls: 新增入口为右下角悬浮圆形加号，脱离文档流并适配底部安全区；搜索框为下划线式输入；状态以文字色区分，不使用厚重标签。
- Reminder summary: 首页顶部主数字显示需要处理的总数，包含已过期和临期；下方用轻量文字分组同时展示「已过期」和「临期」数量。
- Item rows: 首页物品行主标题只显示物品名；次级信息显示「位置 · 到期日」。若用户未设置位置，只显示到期日。
- Item form: 表单顺序为图片、名称、日期、位置、备注；名称和日期必填。日期必须使用日期选择器，不允许自由输入。生产日期 + 保质期区域使用统一的行式字段：标签、值、箭头保持一致字号和颜色，保质期单位使用轻量下拉选择器，不做突兀按钮。位置是自由输入框，历史位置仅作为快捷选择。分类暂不显示。提醒不在 MVP 表单中配置，使用系统默认提醒策略。
- WeChat safe area: 首页顶部必须适配状态栏、刘海屏和微信右上角胶囊操作栏。胶囊所在行的左侧空间可用于家庭名称/标题，右侧必须预留胶囊空间；不要为了避让胶囊把整行都空出来。
- Do not: 不使用渐变背景、大面积彩色卡片、圆角胶囊堆叠、营销式 hero、装饰性图形。
